EMS Handoff Dashboard - Logical Architecture (Development In-Progress)

In high-volume emergency departments, clinicians often operate within a critical information vacuum regarding incoming ambulance traffic. Currently, hospital staff have no visibility into the number of incoming patients or their specific medical complaints unless a patient is "actively coding". This lack of data leads to reactive resource bottlenecks and staffing surges only after patients arrive. I designed this real-time dashboard to eliminate this blind spot, providing the "luxury of anticipation" by transforming incoming dispatch data into actionable insights. This allows clinical leadership to proactively allocate staff and resources, ensuring the department is prepared well before the ambulance reached the emergency department doors.

Logical Architecture and Technical Flow

  • Data Ingestion: Dispatch data is securely processed via Azure API Gateway and Azure Functions, providing an entry point for real-time patient updates.
  • Scalable Persistence: Data is committed to Azure Cosmos DB, partitioned by ED-ID to ensure high performance and strict data isolation across multiple emergency department sites within the hospital system.
  • Real-Time Push Logic: A Change Feed Processor triggers a secondary function upon successful data entry, which utilizes Azure SignalR to push instantaneous updates directly to the hospital-facing dashboard.
  • Outcome: By introducing real-time visibility where an information vacuum previously existed, clinical leadership gains a zero-latency view of incoming volume and acuity - transforming a reactive, blind intake process into a proactive, event-driven workflow.

Technical Stack: Azure Functions, CosmosDB, SignalR, API Gateway, PowerShell, Python
